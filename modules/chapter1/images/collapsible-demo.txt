In this lesson, we are going to look at how to use collapsible blocks within your training content. Collapsible blocks are an excellent way to provide optional information—such as hints, deep dives, or extra context—without cluttering the main lesson flow or distracting the learner. The best part is that this feature is built directly into Asciidoctor, so no additional extensions are required to use it.

First, let’s discuss how to provide helpful hints. You can create a collapsible block to offer guidance that remains hidden until the learner needs it. For example, if you are presenting an exercise, you might include a hint block that encourages learners to try the task on their own first. If they do get stuck, they can open the hint to check that their environment variables are correctly set or to review the previous section on configuration.

Next, consider using collapsible blocks for technical deep dives. These are perfect for explaining complex relationships, such as how specific pipeline stages relate to one another. For instance, in a build process, you might explain that the system runs validate, compile, and then package stages. It is important to note that skipping a stage can leave artifacts out of date, which is why your continuous integration system should always run the full chain. You can reinforce this by showing the command line, such as "./gradlew clean build," to demonstrate the standard execution.

Finally, while most blocks are collapsed by default, you can also create blocks that start in an expanded state. By using the open attribute, you can ensure that the content is visible to the learner as soon as the page loads. Use this sparingly, however, and only when the additional context is optional but helpful enough that you want it visible from the start. 

By strategically using these collapsible elements, you can create a more dynamic and learner-centered experience, ensuring that your documentation remains clean while still providing all the necessary depth for those who need it.
